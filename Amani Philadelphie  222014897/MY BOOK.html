        • EXPLANATION OF TABLES

        1. tsupplier_users

          2. . tsupplier_users
          • user_id (int(11)): A unique identifier for each user. Likely a primary key.
          • username (varchar(50)): Username for logging in to the system.
          • email (varchar(100)): User's email address.
          • password (varchar(255)): User's password for logging in to the system (hashed for security).
        2. . tsupplier_category

          3. • category_id (int(11)): A unique identifier for each supplier category. Likely a primary key.
          • category_name (varchar(50)): Name of the supplier category (e.g., Cleaning Supplies, Electronics).

        3. tsupplier_contactus

          4. • name (varchar(255)): Name of the contact person.
          • email (varchar(255)): Contact's email address.
          • phone_number (varchar(255)): Contact's phone number.
          • message_text (text): Message text from the contact person

        4. . tsupplier_stock

          5. • category_id (int(11)): Foreign key referencing the category_id in the tsupplier_category table.
          • name (varchar(255)): Name of the stock item.
          • description (varchar(255)): Description of the stock item.
          • # price (decimal(10,2)): Price of the stock item.
          • # quantity (int(11)): Quantity of the stock item available.
        5. tsupplier_bookings

          6. • # user_id (int(11)): Foreign key referencing the user_id in the tsupplier_users table (indicates who made the booking).
          • booking_date (date): Date of the booking.
          • details (varchar(255)): Details about the booking.
          Relationships between the tables:
          • A supplier can have many stock items (tsupplier_stock.category_id references tsupplier_category.category_id).
          • A user can make many bookings (tsupplier_bookings.user_id references tsupplier_users.user_id).
